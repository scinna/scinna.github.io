Scinna

Next-generation screenshot tool.

Host your own server to upload your screenshots or pictures, and share them. It's as easy as it gets!

Why Scinna ?

While we love ShareX, by creating Scinna we tried to fix everything that we wanted and did not exist in it.

Cross-platform
Mainly Linux but also supports OS X, Windows, Android and iOS.
Self-hosted
For privacy and preservation purposes. Also features a web-ui to use it without any client.
Your domain
You can use your domain to have clean and professionnal-looking links like https://i.example.com/ImageID.
Plugins !
The toolbox in ShareX is amazing. That's the perfect place to have such small tools but we wanted to be able to extend it.
Also opensource
Of course!

Setting Scinna up

  • Server
  • PC
  • Smartphone

This part will be only useful if you want to use the self-hosted scinna server. If you plan to use the Scinnapse or Scinnamon clients just to take screenshots and saving them, or uploading them to other hosts (Like imgur, etc...) you don't need to follow this.

This tutorial expects you to use a Linux server and have some basic Linux knowledge. If you are using anything else, you are on your own. Due to the nature of the code, it will probably work on OSX and require a few tweaks on Windows. This will be solved in the future.


Requirements
  • A linux server (Dedicated, VPS, Raspberry pi, etc...)
  • The nginx webserver installed (Used as a reverse proxy)
  • A (sub-)domain name that links to the server's IP
  • (Optional, but preferred) A SSL certificate (Let's Encrypt)
  • (Optional, but preferred) A database server running and with an account + database created for scinna
  • (Optional, but preferred) A SMTP server for account validation and password recovery
  • 20 minutes

Connect via SSH to your webserver.

We'll create a user for the scinna instance, no need to set him a password: 

$ useradd -m scinna

Lets download the server executable and create service to keep it running:

$ cd /tmp
$ wget https://TMP_URL_TO_REPLACE/ -o /usr/bin/scinna
$ chmod +x /usr/bin/scinna
$ scinna -gensysd > /etc/systemd/system/scinna
$ systemctl enable --now scinna

Scinna is now running, let's set up the reverse-proxy now. We're using nginx.

Here's a simple virtualhost just change your hostname and you're ready to go. Don't forget to restart the nginx service afterward.

server {
	listen 80;
	server_name i.example.com;
	return 301 https://$host$request_uri;
}

server {
	listen 443 ssl http2;
	server_name i.example.com;
	include snippets/ssl.conf;
	error_log  /home/scinna/error.log warn;
	access_log /home/scinna/access.log combined;

	client_max_body_size 5M; # You might want to change that depending on what size of picture you'll be uploading.

	location / {
		proxy_set_header Host $host;
		proxy_set_header X-Real-IP $remote_addr;
		proxy_pass http://127.0.0.1:1635;
	}
}

You're close to be done!

Just restart nginx, go to the specified domain name and follow the setup. It will lead you to the database initialization, account creation and server settings.

Scinna is available for Windows, Linux and OS X.

Download the software for your operating system and set it up as you would with any other software.

For ArchLinux, you can find the scinnapse package in the AUR.


When you'll first run Scinnapse, it will ask you to login with your Scinna account.

If you don't have a Scinna server you can click on "No, thanks" and it will work with the scinna features disabled.

The Scinnamon app is a tool built to help you share picture from your smartphone.

Once you installed it, and logged into your scinna account, you can use the standard share feature of your smartphone and share it to Scinnamon.

It will ask you the title / description that you'd like and the link will be copied to your clipboard.


You'll be able to download this app from the Google Play Store and F-Droid for Android.

Unfortunately, since I develop only this app for mobile, I have no interest in having a 99$ yearly cost for Apple developer account so no luck there.

I'm no iOS expert, but you can maybe install the ipa with the AltStore and it's patched AltServer (Windows / OS X) on non-jailbroken devices. Jailbroken users should have no problem installing it themselves.
